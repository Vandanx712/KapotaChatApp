Generate the frontend UI for a modern, premium real-time chat web application named “Kapota(pigeon)ChatApp”.

This instruction applies to the entire UI and must be followed strictly.

The goal is to create a calm, classic, and premium chat experience that feels timeless and professional — NOT trendy or flashy.

--------------------
TECH STACK RULES
--------------------

- Use React with JavaScript only.
- Do NOT use TypeScript.
- Use Vite + React for project setup.
- Use react-router-dom for routing.
- Use Tailwind vite v4  for general styling.
- Use react-icons for icon library 

Do NOT use:
- Next.js
- Material UI
- Ant Design
- Bootstrap
- Any other UI library
- Inline styles unless absolutely unavoidable


--------------------
DESIGN & VIBE
--------------------

- Overall tone: calm, classic, premium, and mature.
- Avoid common blue / indigo chat app themes.
- UI should feel suitable for long conversations.
- Clean spacing, soft shadows, rounded corners.
- Inspired by high-end productivity tools and luxury SaaS apps.


--------------------
COLOR SYSTEM (STRICT)
--------------------

Primary Color (Brand):
- Deep Forest Green (#0F3D2E)

Secondary Background:
- Soft Cream (#FAF7F2)

Accent Color:
- Espresso Brown (#6B4F3F)

Text Colors:
- Primary text: near-black charcoal
- Muted text: soft gray

Usage Rules:
- Forest Green: primary actions, sent messages, active states
- Cream: main backgrounds and chat panels
- Espresso: subtle highlights, borders, secondary buttons
- Do NOT introduce blue or neon colors

--------------------
TYPOGRAPHY
--------------------

Primary Font:
- Inter (used everywhere for readability)

Accent Font:
- Playfair Display (ONLY for logo or page titles)

Rules:
- Chat text must be clean and readable
- Avoid overly bold or playful fonts
- Maintain consistent font hierarchy


--------------------
ANIMATION & INTERACTIONS
--------------------

- Use subtle, premium micro-interactions only
- No bounce, spring, or playful motion
- Prefer ease-out transitions

Examples:
- Message slide-in: 6–12px vertical movement
- Button hover: slight scale (1.02)
- Input focus: soft glow or border transition
- Sidebar transitions: smooth 200–300ms

Animations should feel invisible but polished

--------------------
RESPONSIVENESS
--------------------

- Fully responsive design
- Desktop-first layout
- Must adapt cleanly for tablet and mobile
- Chat layout should remain usable on small screens

--------------------
COMPONENT PHILOSOPHY
--------------------

- Keep UI simple and readable
- Avoid visual clutter
- Focus on usability over decoration
- No unnecessary UI elements

LAYOUT STRUCTURE
--------------------

  1. Top navigation bar

--------------------

--------------------

- Display the following tabs horizontally:

  - Status
  - Explore
  - Video Call

- Active tab highlighted using Forest Green accent.
- Inactive tabs use muted text color.

UI Rules:
- Clean tab style (no heavy borders).
- Smooth underline or indicator animation when switching tabs.
- Keep spacing balanced and readable.
